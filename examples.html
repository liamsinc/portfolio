Code Snippet 1 - Portfolio (app.js)

This function is called whenever the page loads. It's used to apply styles to the navigation links, uncheck the side menu button checkbox, clear form input fields and runs a function which determines if the header scroll button should be visible. It determines the current page based on a URL ending with a specific string.

I'm using jQuery to select the document and attaching an anonymous arrow function to the ready event. First, I grab the URL of the current page and store it in the constant currentURL. I then run the constant through a conditional. The first conditonal checks to see if URL ends with anything which indicates index.html is the page currently being used. This is followed by further conditionals (assuming the first conditonal evaluates to false) which check for URLs ending with about.html, examples.html or scs.html.

The reason why I have 6 statements in the first conditional is to account for a number of issues that arise when the page loads from differnt sources (local server, github pages, cPanel domain). It also addresses the issue of the "My Portfolio" and "Contact Me" navigation links being page-local hyperlinks. In this case, the main logo would be the active link as the portfolio section and contact form reside on the homepage (index.html).

If for some reason none of the conditions are met, the function will log a message to the JS console. I used a string literal to print out the value stored in currentURL. While I was building and testing this function, it allowed me to quickly identify additional strings I needed add to my first if statement. Due to the iterative building process, the function now works whether I launch the site via VSC's live server extension, through GitHub Pages or my cPanel domain provided by Netmatters.

Code Snippet 2 - Portfolio (app.js)

This function is called whenever the browser window is resized. It's used to fix site-breaking behaviour and manipulates the side menu button and the vertical header. It then runs a function which determines if the header scroll button should be visible.

While building my website in responsive mode using my browser's developer tools, I noticed after I implemented the side menu on mobile/tablet devices, if the user was to somehow expand their browser window with the side menu open, it would not show the vertical header when the medium breakpoint was reached. The medium breakpoint triggers when the window width exceeds 768px, and usually this would show the vertical header and hide the side menu button as it's no longer needed.

To fix this issue, I wrote this function. The function is invoked when the browser window is resized. First, it checks if the window width is greater than or equal to 768px. If this is true, it unchecks the checkbox attached to the side menu button and shows the vertical header. If it is false, it then checks if the window width is less than 768px and if the checkbox (menu button) is not checked. If this statment returns true, it hides the header.
